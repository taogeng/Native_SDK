367 /*!*********************************************************************************************
368 \brief Return if the format is Normalized (represents a range between 0..1 for unsigned types
369  or between -1..1 for signed types)
370 \param type The format to test.
371 \return True if the format is Normalised.
372 \description A Normalised format is a value that is stored as an Integer, but that actually
373 represents a value from 0..1 or -1..1 instead of the numeric value
374 of the integer. For example, for a normalised unsigned byte value, the value
375 0 represents 0.0, the value 127 represents 0.5 and the value 255 represents 1.0.
376 ***********************************************************************************************/
377 inline bool isNormalised(Enum type)
378 {
379  return (type == DataType::Int8Norm || type == DataType::UInt8Norm
380  || type == DataType::Int16Norm
381  || type == DataType::UInt16Norm);
382 }
