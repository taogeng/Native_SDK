312  /*!***********************************************************************
313  \param[in] bumpScale Floating point "height" value to scale the bump map.
314  \param[in] bumpOrder Up to 4 character string, with values x,y,z,h in
315  some combination.
316  \brief Sets a texture's bump map data.
317  \description For \p bumpOrder Not all values need to be present.
318  Denotes channel order; x,y,z refer to the corresponding axes,
319  h indicates presence of the original height map. It is possible
320  to have only some of these values rather than all.
321  For example if 'h' is present alone it will be considered a height map.
322  The values should be presented in RGBA order, regardless
323  of the texture format, so a zyxh order in a bgra texture
324  should still be passed as 'xyzh'. Capitals are allowed.
325  Any character stored here that is not one of x,y,z,h
326  or a NULL character will be ignored when PVRTexLib
327  reads the data, but will be preserved. This is useful
328  if you wish to define a custom data channel for instance.
329  In these instances PVRTexLib will assume it is simply
330  color data.
331  *************************************************************************/
332  void setBumpMap(float bumpScale, string bumpOrder);
