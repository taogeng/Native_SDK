35 /*!*********************************************************************************************************************
36 \brief The Model class represents an entire Scene, or Model. It is mainly a Node structure, allowing various different
37  kinds of data to be stored in the Nodes.
38  The class contains a tree-like structure of Nodes. Each Node can be a Mesh node (containing a Mesh), Camera
39  node or Light node. The tree-structure assumes transformational hierarchy (as usual).
40  Transformations are expressed through Animation objects (a static transform is an animation with a single frame)
41  There is an implicit order in the nodes - First in the array the Mesh nodes will be laid out, then Camera and
42  Light nodes.
43 ***********************************************************************************************************************/
44 class Model : public Asset<Model>

48  /*!*********************************************************************************************************************
49  \brief The Node represents a Mesh, Camera or Light.
50  A Node has its own parenting, material, animation and custom user data. The tree-structure assumes
51  transformational hierarchy (as usual), so parent transformations should be applied to children.
52  Transformations are expressed through Animation objects (a static transform is an animation with a single
53  frame).
54  Note: Node ID and MeshID can sometimes be confusing: They will always be the same (when a MeshID makes sense)
55  because Meshes are always laid out first in the internal list of Nodes.
56  ***********************************************************************************************************************/
