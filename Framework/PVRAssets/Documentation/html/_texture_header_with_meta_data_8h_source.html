151  /*!***********************************************************************
152  \param[in] bumpScale Floating point "height" value to scale the bump map.
153  \param[in] bumpOrder Up to 4 character string, with values x,y,z,h in
154  some combination. Not all values need to be present.
155  Denotes channel order; x,y,z refer to the
156  corresponding axes, h indicates presence of the
157  original height map. It is possible to have only some
158  of these values rather than all. For example if 'h'
159  is present alone it will be considered a height map.
160  The values should be presented in RGBA order, regardless
161  of the texture format, so a zyxh order in a bgra texture
162  should still be passed as 'xyzh'. Capitals are allowed.
163  Any character stored here that is not one of x,y,z,h
164  or a NULL character will be ignored when PVRTexLib
165  reads the data, but will be preserved. This is useful
166  if you wish to define a custom data channel for instance.
167  In these instances PVRTexLib will assume it is simply color data.
168  \brief Sets a texture's bump map data.
169  *************************************************************************/
170  void setBumpMap(float bumpScale, std::string bumpOrder);
